## What is Golden Codex?

Golden Codex transforms your images into **museum-grade digital artifacts** with:

- **AI Metadata Enrichment** (Nova) -- 20+ structured metadata sections generated by Gemini AI
- **4K Upscaling** (Flux) -- ESRGAN-powered enhancement with 17 specialized models
- **Metadata Infusion** (Atlas) -- Embed provenance data directly into image files (XMP/IPTC)
- **Hash Registration** -- Perceptual hash + Soulmark (SHA-256) for provenance verification
- **Permanent Storage** -- Optional Arweave archiving for 200+ year preservation

Perfect for: digital art platforms, NFT marketplaces, photography workflows, archive digitization, and AI training dataset creation.

## Quick Start

### 1. Get your API key

Sign up at [golden-codex.com](https://golden-codex.com), open **Studio**, go to **Settings > API** tab, and generate an API key.

### 2. Upload an image

```bash
# Get a signed upload URL (no charge)
curl -X POST https://nova-agent-172867820131.us-west1.run.app/upload-url \
  -H "Content-Type: application/json" \
  -H "X-API-Key: gcx_live_YOUR_KEY_HERE" \
  -d '{"filename": "my_artwork.jpg", "content_type": "image/jpeg"}'
```

Response:
```json
{
  "status": "success",
  "upload_url": "https://storage.googleapis.com/...",
  "image_url": "gs://the-golden-codex-1111.firebasestorage.app/api/USER_ID/1234_my_artwork.jpg",
  "expires_in": 900
}
```

Upload your file directly to GCS (bytes go straight to storage, not through our servers):
```bash
curl -X PUT -H "Content-Type: image/jpeg" \
  --upload-file my_artwork.jpg \
  "THE_UPLOAD_URL_FROM_ABOVE"
```

### 3. Enrich your image

```bash
curl -X POST https://nova-agent-172867820131.us-west1.run.app/enrich \
  -H "Content-Type: application/json" \
  -H "X-API-Key: gcx_live_YOUR_KEY_HERE" \
  -d '{
    "image_url": "gs://the-golden-codex-1111.firebasestorage.app/api/USER_ID/1234_my_artwork.jpg",
    "parameters": {
      "analysis_depth": "full",
      "content_type": "artwork"
    }
  }'
```

Response:
```json
{
  "status": "success",
  "golden_codex": {
    "schemaVersion": "1.5-STUDIO",
    "title": "Crimson Tide at Dusk",
    "description": "A meditation on the boundary between day and night...",
    "coreIdentity": { "..." },
    "visualAnalysis": { "..." },
    "artisticStatement": { "..." },
    "emotionalContent": { "..." },
    "contextualGraph": { "..." },
    "creativeContent": { "..." },
    "ownershipAndRights": { "..." },
    "_identifiers": {
      "soulmark": "3b1cbec224a19be..."
    }
  },
  "processing_time_ms": 12345
}
```

## Authentication

All API requests require your key in the `X-API-Key` header:

```
X-API-Key: gcx_live_YOUR_KEY_HERE
```

Generate keys in **Studio > Settings > API** tab. Each key is SHA-256 hashed server-side -- we never store your raw key. You can have up to 5 active keys per account.

### `POST /upload-url` (Nova)

Get a signed URL for direct-to-GCS image upload. **No charge.**

**Request:**
```json
{
  "filename": "my_artwork.jpg",
  "content_type": "image/jpeg"
}
```

**Response:**
```json
{
  "status": "success",
  "upload_url": "https://storage.googleapis.com/...",
  "image_url": "gs://bucket/api/user_id/timestamp_filename",
  "content_type": "image/jpeg",
  "expires_in": 900
}
```

**Workflow:** Call this endpoint, then `PUT` your file to `upload_url`, then pass `image_url` to `/enrich` or `/upscale`.

---

### `POST /enrich` (Nova)

AI metadata enrichment powered by Gemini.

**Request:**
```json
{
  "image_url": "gs://... or https://...",
  "parameters": {
    "analysis_depth": "full",
    "content_type": "artwork"
  }
}
```

| Parameter | Values | Default | Description |
|-----------|--------|---------|-------------|
| `image_url` | GCS path or HTTPS URL | required | Image to analyze |
| `image_data` | base64 string | -- | Alternative: base64-encoded image bytes |
| `mime_type` | `image/jpeg`, `image/png`, etc. | -- | Required if using `image_data` |
| `parameters.analysis_depth` | `standard`, `full` | `standard` | Standard (2 GCX) or Full GCX (3 GCX) |
| `parameters.content_type` | `artwork`, `photo`, `commercial` | `artwork` | Tailors analysis style |

**Response:**
```json
{
  "status": "success",
  "golden_codex": { "..." },
  "processing_time_ms": 12345
}
```

The `golden_codex` object contains 20+ structured sections including `coreIdentity`, `visualAnalysis`, `artisticStatement`, `emotionalContent`, `contextualGraph`, `creativeContent`, `ownershipAndRights`, and `_identifiers` (with Soulmark hash).

### `POST /upscale` (Flux)

ESRGAN AI upscaling with 17 specialized models.

**Request:**
```json
{
  "image_url": "gs://... or https://...",
  "user_id": "your_user_id",
  "parameters": {
    "upscale_factor": 4,
    "model": "realesrgan_x4plus"
  }
}
```

| Parameter | Values | Default | Description |
|-----------|--------|---------|-------------|
| `image_url` | GCS path or HTTPS URL | required | Image to upscale |
| `parameters.upscale_factor` | `2`, `4` | `4` | Upscale multiplier |
| `parameters.model` | See model list below | `realesrgan_x4plus` | ESRGAN model |

**Available Models:**

| Model | Scale | Best For |
|-------|-------|----------|
| `realesrgan_x2plus` | 2x | General-purpose fast upscale |
| `realesrgan_x4plus` | 4x | General-purpose maximum quality |
| `realesrgan_x4plus_anime` | 4x | Anime and illustration |
| `real_esrgan_x4_v3` | 4x | Photography |
| `ultrasharp_x4` | 4x | Maximum detail preservation |
| `remacri_x4` | 4x | Faces and portraits |
| `nmkd_siax_x4` | 4x | Balanced quality/speed |
| `foolhardy_x4` | 4x | Digital art |
| `lollypop_x4` | 4x | Vibrant colors |

**Response:**
```json
{
  "status": "success",
  "upscaled_image_url": "gs://...",
  "signed_url": "https://... (24h expiration)",
  "processing_time_ms": 3400,
  "original_size_bytes": 1234567,
  "upscaled_size_bytes": 4567890
}
```

### `POST /infuse` (Atlas)

Embed Golden Codex metadata into image files using XMP standards. Includes perceptual hash registration for provenance verification.

**Request:**
```json
{
  "image_url": "gs://... or https://...",
  "user_id": "your_user_id",
  "golden_codex": { "..." },
  "parameters": {
    "register_hash": true
  }
}
```

**Response:**
```json
{
  "uuid": "a7f3c8d9-...",
  "soulmark": "3b1cbec224a19be...",
  "final_url": "https://storage.googleapis.com/.../final.png",
  "artifact_id": "GCX00123"
}
```

Atlas infusion is **free (0 GCX)** when called via API key -- it's bundled with Nova enrichment.

## Full Pipeline Example

Process an image through the complete Golden Codex pipeline:

```bash
API_KEY="gcx_live_YOUR_KEY_HERE"
NOVA="https://nova-agent-172867820131.us-west1.run.app"
FLUX="https://flux-agent-172867820131.us-west1.run.app"
ATLAS="https://atlas-agent-172867820131.us-west1.run.app"

# 1. Upload (free)
UPLOAD=$(curl -s -X POST "$NOVA/upload-url" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_KEY" \
  -d '{"filename": "artwork.jpg", "content_type": "image/jpeg"}')

UPLOAD_URL=$(echo $UPLOAD | jq -r '.upload_url')
IMAGE_URL=$(echo $UPLOAD | jq -r '.image_url')

curl -X PUT -H "Content-Type: image/jpeg" --upload-file artwork.jpg "$UPLOAD_URL"

# 2. Enrich with Nova Full GCX (3 GCX)
ENRICHED=$(curl -s -X POST "$NOVA/enrich" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_KEY" \
  -d "{\"image_url\": \"$IMAGE_URL\", \"parameters\": {\"analysis_depth\": \"full\"}}")

CODEX=$(echo $ENRICHED | jq '.golden_codex')

# 3. Upscale with Flux (2 GCX)
UPSCALED=$(curl -s -X POST "$FLUX/upscale" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_KEY" \
  -d "{\"image_url\": \"$IMAGE_URL\", \"user_id\": \"YOUR_USER_ID\", \"parameters\": {\"upscale_factor\": 4}}")

UPSCALED_URL=$(echo $UPSCALED | jq -r '.upscaled_image_url')

# 4. Infuse metadata into upscaled image (0 GCX -- bundled)
FINAL=$(curl -s -X POST "$ATLAS/infuse" \
  -H "Content-Type: application/json" \
  -H "X-API-Key: $API_KEY" \
  -d "{\"image_url\": \"$UPSCALED_URL\", \"user_id\": \"YOUR_USER_ID\", \"golden_codex\": $CODEX}")

echo "Soulmark: $(echo $FINAL | jq -r '.soulmark')"
echo "Final: $(echo $FINAL | jq -r '.final_url')"

# Total cost: 5 GCX (3 + 2 + 0)
```

## Using `image_data` (Base64) Instead of URLs

If you prefer not to use the upload URL flow, Nova accepts base64-encoded images directly:

```bash
BASE64=$(base64 -w0 my_artwork.jpg)

curl -X POST https://nova-agent-172867820131.us-west1.run.app/enrich \
  -H "Content-Type: application/json" \
  -H "X-API-Key: gcx_live_YOUR_KEY_HERE" \
  -d "{
    \"image_data\": \"$BASE64\",
    \"mime_type\": \"image/jpeg\",
    \"parameters\": {\"analysis_depth\": \"full\"}
  }"
```

This sends the image bytes through the API server. For large images, the signed upload URL approach is recommended.

## Error Handling

All errors return a consistent format:

```json
{
  "status": "error",
  "message": "Insufficient token balance"
}
```

| HTTP Code | Meaning |
|-----------|---------|
| `400` | Bad request (missing required fields) |
| `401` | No API key provided |
| `403` | Invalid API key or insufficient credits |
| `500` | Server error |
